# Active Reading Strategies (SQ3R, Three-Pass) Translated to Agent Reading Patterns Over Retrieved Chunks

**Scope.** Round-1 settled *single-query* mechanics on the librarian (verbatim query > rewrite/HyDE; k=20 best, k=8 value point; quote-first generation; abstention contract 12%->0%). This note takes the human *active-reading* literature — SQ3R (Robinson, 1946) and Keshav's three-pass method — as a source of **structured, multi-phase reading protocols** and asks how each phase maps onto an agent reading librarian chunks, and how the protocol should differ by task family.

## 1. SQ3R and three-pass are the same idea: progressive, question-driven, multi-pass reading

SQ3R = **Survey, Question, Read, Recite, Review**. The load-bearing claims from the education literature: it converts *passive* reading (eyes over text, no engagement) into *active* reading, where the reader (a) skims structure first, (b) turns headings into questions, (c) reads *to answer those questions*, (d) recites/recalls without the text, (e) reviews. Quasi-experimental pre/post studies report statistically significant comprehension and retention gains over rereading (effect sizes "large"; one university study reports a large effect size, another a paired t-test p<0.001), strongest "when applied systematically in accordance with established steps." Keshav's **three-pass method** (ACM SIGCOMM CCR 37(3), 2007) is the research-paper analogue with explicit budgets: **Pass 1** (5-10 min: title/abstract/intro/headings/conclusions — a triage/skim that answers "is this worth my time?"), **Pass 2** (~1 hr: figures, evidence, mark references — "grasp content, not details, summarise in your own words"), **Pass 3** (4+ hr: reconstruct/replicate). Both frameworks encode the same three primitives the agent literature has independently rediscovered: **skim-before-deep-read**, **question-conditioned reading**, and **recall-to-consolidate**.

## 2. Survey/Pass-1 maps to gist-memory and coarse-to-fine retrieval (and it pays)

The "Survey" step (skim structure, build a gist) is exactly **ReadAgent** (Lee et al., arXiv:2402.09727): paginate the document into episodes, compress each into a short *gist memory*, then **interactively look up** the original passages only when a detail is needed. It "extends the effective context length 3.5-20x" and beats the best retrieval baseline on NarrativeQA by **+12.97% LLM-rating and +31.98% ROUGE-L** (71k-word avg, 343k max). **LATTICE** (arXiv:2510.13217) and **GraphSkill** (arXiv:2603.06620) make the same move structurally: impose a semantic tree / summary hierarchy and let the LLM *traverse top-down*, pruning irrelevant branches early to hit task-relevant entries "with only a few LLM queries" (LATTICE: logarithmic search complexity). **CAFE** (arXiv:2505.10063) frames it as "coarse-to-fine information seeking," motivated by the same over-retrieval failure we hit: "performance significantly declines as the number of additional input documents increases." For the librarian this is a concrete pattern: a *survey pass* = read only the **breadcrumb/heading layer** of the top-k chunks (cheap, gisty) before deciding which chunks to read in full — a within-result-set skim that the current "read all 20 chunks" flow skips.

## 3. Question/Pass-2 maps to self-ask — powerful but drift-prone

The "Question" step (turn headings into questions, read to answer them) is **Self-Ask** (Press et al., 2023) and its interleaved cousins (ReAct, IRCoT): generate explicit sub-questions, route each to retrieval, answer, repeat. This is the documented winner for *multi-hop* work where "information needed for later steps depends on intermediate deductions" — single-pass retrieval cannot resolve the dependency. But the failure mode is sharp and directly relevant to a task-conditioned policy: purely self-guided iterative questioning suffers **semantic drift and error propagation** — "as retrieval iterations increase, generated queries gradually drift from the correct reasoning path, and noisy information accumulates," and self-reflection is "an unreliable, ungrounded signal" because the same model that errs also judges the error (motivation for executable PyRAG, structured RT-RAG/Reasoning-in-Trees arXiv:2601.11255). Translation: the *question-generation* reading mode is the right protocol for **synthesis and multi-hop science** but needs a drift guard (verbatim-anchored sub-queries, per Round-1; a cap on hop count; structured rather than free-form decomposition).

## 4. Recite/Review maps to free-recall synthesis-notes — the strongest, cheapest add-on

The "Recite" + "Review" steps have the largest, best-quantified evidence base of any active-reading component. **Retrieval practice / testing effect**: free-recall after reading beats rereading for retention with meta-analytic effect sizes **g=0.50 (Rowland 2014, 159 effects)** and **g=0.61 (Adesope et al. 2017)**; *generative* formats (free recall, g=0.81) beat *recognition* formats (multiple choice); the benefit grows "as initial retrieval success increases, especially >75%." The agent analogue is a **recite pass**: after reading the retrieved chunks, the agent writes a from-memory summary of what it learned *with citations*, then a **review pass** re-queries to verify/fill gaps — i.e. the scratchpad-synthesis + claim-verify loops covered in sibling notes 26/27, but now grounded in the testing-effect literature rather than just engineering folklore. The ">75% initial retrieval success" moderator is a usable signal: if the recite pass is mostly gaps, *the read was insufficient* — re-retrieve, don't synthesise.

## 5. Task-conditioning: which passes to run, and how deep

The two frameworks already encode a depth dial; the task selects where to stop. **Lookup (maths fact, API signature, single detector parameter):** Pass-1 only — survey the breadcrumb layer of one search, read the single best chunk, hard-stop (Keshav: "first pass gives a general idea"; SQ3R degenerates to S+single-Q). No recite, no review. **Science/multi-hop QA:** Question-driven Pass-2 with self-ask sub-queries + drift guard. **Literature synthesis / 4D-STEM requirements recovery:** full SQ3R — survey many sources shallow, question-decompose, read-to-answer, **recite a cited synthesis from memory**, review/re-query to close gaps; this is the only family that earns Pass-3-style reconstruction. **Coding/debugging:** Pass-1 triage then depth-first Pass-2 on the one relevant API/source chunk; recite = restate the contract before using it. **Learning/tutoring:** the recite/free-recall step is the *product*, not an add-on — the g=0.81 free-recall result says the agent should make the *user* (or itself, when self-teaching) recall before revealing the chunk.

## 6. Actionable implications for task-conditioned librarian experiments

1. **Add a "survey pass" as a measurable mode.** Read only breadcrumbs/headings of the k=20 set first, select a subset to read in full, then answer. Benchmark answer quality and tokens-read against the current read-all-20 flow (hypothesis from ReadAgent/CAFE: equal or better quality, far fewer tokens, fewer "lost-in-the-middle" distractions).
2. **Operationalise SQ3R's "Question" step as guarded self-ask, gated by task.** Enable sub-question decomposition for synthesis/multi-hop only; for lookup, force single verbatim query (Round-1). Cap hops and anchor each sub-query to verbatim spans to suppress the documented semantic drift; measure drift as sub-query divergence from the original.
3. **Add a "recite pass" and use recall-success as a stop/continue signal.** After reading, the agent writes a cited from-memory summary; if cited recall covers <~75% of the question's facets, re-retrieve rather than answer (testing-effect moderator). Compare answer correctness and citation precision with/without the recite pass.
4. **Map the three Keshav budgets to three librarian tiers and benchmark per task.** Pass-1 = 1 search, breadcrumb-only; Pass-2 = self-ask, full-chunk reads, ~1.26-search baseline (PaperQA2); Pass-3 = multi-search reconstruction (synthesis only). Measure where each task's quality saturates to find its natural pass-depth.
5. **Treat free-recall as the deliverable for the learning task family.** For tutoring/self-teaching, withhold the chunk and elicit recall first (g=0.81 generative-format result); measure retention/accuracy vs. chunk-first reading.
6. **Instrument the active-vs-passive contrast directly.** The whole SQ3R claim is active > passive reading. The testable agent version: question-conditioned reading (read each chunk *against* a held sub-question) vs. undirected "read all chunks then answer." Predict the active mode wins on multi-hop/synthesis and is neutral on lookup — locating exactly where the protocol overhead is justified.
