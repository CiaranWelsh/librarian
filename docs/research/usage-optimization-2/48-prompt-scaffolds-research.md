# System-Prompt Scaffolds for Research/Synthesis with Tools: Structured Workflows, Checklists, and Measured Effects on Thoroughness

## 1. The scaffold is the only lever: prompt tokens *are* the control program

A system prompt is not executed like code — the tokens condition the next-token distribution on every forward pass, and the "plan" is whatever the model emits under that conditioning. There is no separate planning engine, so the structure written into the prompt directly shapes the sequence of tool calls and the thoroughness of the result. Anthropic states this bluntly: because each agent is steered by a prompt, "prompt engineering was our primary lever." Their uncontrolled baseline failed in exactly the ways a librarian agent would — spawning 50 subagents for trivial queries, "scouring the web endlessly for nonexistent sources," and defaulting to "overly long, specific queries that return few results." The fix was *not* a smarter model but a more structured prompt. This is the central premise for Round-2: holding retrieval mechanics fixed (Round-1's verbatim query, k=20, quote-first, abstention contract), the scaffold is what's left to optimize, and it is task-conditioned.

## 2. Effort-scaling rules are the highest-value scaffold component — with hard numbers

Anthropic's strongest reported finding is that *embedding explicit effort budgets per task type* fixed the over/under-investment failure. Their prompt encodes: simple fact-finding → **1 agent, 3–10 tool calls**; direct comparison → **2–4 agents, 10–15 calls each**; complex research → **10+ agents**. The justification: "agents struggle to judge appropriate effort … so we embedded scaling rules in the prompts." The payoff is large in aggregate — the multi-agent (scaffolded, parallel) system beat single-agent Opus 4 by **90.2%** on internal research eval — but the load-bearing diagnostic is that **token usage alone explained ~80% of performance variance** on BrowseComp, with tool-call count and model choice covering the rest (95% total). This says thoroughness is *bought with search volume*, and the scaffold's job is to spend that budget proportionally to task difficulty rather than uniformly. For a single-agent librarian, the directly transferable artifact is a search-count budget table keyed to task family (lookup vs synthesis vs investigation), which is sibling-37's decomposition cap expressed as a prompt rule.

## 3. Reason-before-act and broad→narrow scaffolds raise thoroughness; the gains are measured

Two scaffold patterns recur across the strongest systems. (a) **ReAct's Thought→Action→Observation loop** forces explicit reasoning before each tool call; the AgentSynth study shows a planning + tool-aware-reasoning + self-verification scaffold (Agent S3) roughly *doubles* success on weaker backbones (o4-mini, GPT-4.1) and lifts strong ones, confirming the scaffold contributes orthogonally to model quality. (b) **"Start wide, then narrow"** — Anthropic explicitly instructs agents to begin with short broad queries, survey the landscape, then drill in, "mirror[ing] how expert human researchers work," because the default failure is premature over-specific queries. Both are cheap prompt edits with measured effect, and both map cleanly onto the librarian: a per-search interleaved-reasoning step ("what does this chunk's breadcrumb tell me, what gap remains") plus a broad-first query trajectory.

## 4. Checklist / verification scaffolds reduce *omission*, not misunderstanding — and can backfire

The thoroughness gains from checklist-style verification are real but narrow. Chain-of-Verification (draft → plan verification questions → answer them independently → revise) **more than doubled precision (0.17→0.36)** and cut hallucinated entities **2.95→0.68** on a Wikidata list task while preserving correct content, and lifted FActScore biography precision **+28%** and MultiSpanQA F1 **+23%**; critically, the *factored* variant (verifier blind to the draft) beats the joint one (63.7→71.4 FActScore), because attending to the draft re-copies its errors. Reflexion-style modules show even larger task gains in agentic settings (+13.7 / +32.6 pp in one ablation). But the literature is *not* uniformly positive: on pure planning/reasoning tasks, ungrounded self-critique **degrades** quality and worsens with more rounds (Stechly et al.; Huang et al. survey: no general-task self-correction works without *external* feedback). The lesson for the librarian: a verification checklist helps because the corpus *is* the external grounding (re-query to confirm each claim against a fresh chunk), and it should be **factored** (verify against newly retrieved chunks, not the draft). This is sibling-27 made concrete.

## 5. Over-structuring has a measured cost: format constraints degrade reasoning

Scaffolds are not free. Tam et al. show strict output-format constraints (JSON-mode constrained decoding) cause **10–30% reasoning degradation**, because the model is forced to emit answer fields before finishing its chain-of-thought; CRANE gives the theoretical account. A 2025 causal-inference reanalysis tempers this — *no* causal effect in 43/48 scenarios under controlled settings, and reasoning models (o3) are far more format-robust — but the practical rule stands: **let the model reason in prose, impose structure only at the end** (DECO-G's decoupling). For the librarian this means the scaffold should structure the *process* (search budget, broad-first, quote-first, verify) without forcing rigid JSON mid-reasoning on synthesis tasks; rigid structure is safe for atomic lookups, risky for multi-hop synthesis.

## 6. Actionable implications for task-conditioned librarian experiments

1. **A/B a structured scaffold vs a bare instruction, per task family.** Hypothesis (§1–2): the scaffold's win *grows with task compositionality* and is near-zero for atomic lookup. Measure answer correctness, total searches, and distinct-source coverage; predict crossover at the single-fact case (Round-1's verbatim-single-query regime).
2. **Encode an effort-budget table as the core scaffold and sweep it.** Lookup → 1 search; comparison → 2–4; synthesis/investigation → broad-first trajectory of ~5–8 (sibling 37's IterDRAG ≤5 cap, file 19's knee). Because token/search volume drove ~80% of variance, this is the highest-expected-value single experiment.
3. **Test a *factored* verification checklist gated by the abstention contract.** After drafting, emit one verification query per load-bearing claim, retrieve *fresh* chunks (not the draft), and require a confidence label. Predict gains on synthesis recall/precision (CoVe-style) but *no* gain — possibly harm — on tasks lacking corpus grounding (§4); use abstain/low-confidence as the trigger.
4. **Ablate scaffold components individually** (effort budget vs broad-first vs reason-before-search vs verify-checklist) rather than as a bundle — ablations are what separate real drivers (cf. self-contrast: removing checklist generation costs 2–4 pp). Report which component carries the thoroughness gain.
5. **Bound the structure: prose-reason, late-format.** For synthesis prompts, do not force JSON mid-reasoning (§5's 10–30% risk); compare a rigid-format scaffold against a process-only scaffold on multi-hop questions to confirm the cost exists on *our* corpus and model.
6. **Instrument plan/checklist coverage as a leading quality indicator** (per sibling 37 / STORM): score the scaffold-induced sub-question or verification-item list against the gold answer's required facets; if coverage predicts final correctness, it becomes a cheap early gate before spending the full search budget.
