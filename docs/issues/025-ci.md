# 025 — CI (GitHub Actions + Qdrant service container)

**Phase:** J (post-v1) · **Status:** Open · **Deps:** none

## Goal

Catch drift. Today tests run on demand only — nothing prevents a regression from sliding into `main`. A minimal GitHub Actions workflow runs the full test suite against a Qdrant service container.

## Acceptance criteria

- `.github/workflows/ci.yml` runs on every push and PR to `main`.
- Steps: `cargo fmt --check`, `cargo clippy -- -D warnings`, `cargo build`, `cargo test --workspace --no-fail-fast`.
- Qdrant runs as a service container on the runner (port 6533 to match the test default — or expose the port via env so tests pick it up).
- Caching: `Swatinem/rust-cache@v2` or equivalent; cold builds shouldn't dominate.
- Build matrix: at minimum `ubuntu-latest`. macOS optional (the user dev's there but CI doesn't need it).
- Badge in README pointing at the workflow's status.

## Test plan

- Land the workflow, confirm it goes green on a no-op push.
- Confirm a deliberate `assert!(false)` in any test turns the badge red.

## Notes

API-key-gated tests (live OpenAI / Voyage smoke) stay opt-in via env vars; they should not run in CI by default.
