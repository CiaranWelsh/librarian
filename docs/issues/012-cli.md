# 012 — CLI commands

**Phase:** E · **Status:** Open · **Deps:** 011

## Goal

The `librarian` CLI binary with the v1 commands wired against the real adapters. Backs **F-7.1, F-7.2, F-7.4**.

## Acceptance criteria

- `librarian` binary in `crates/cli`. Subcommands: `ingest`, `remove`, `status`, `snapshot`, `restore`, `start`, `stop`, `restart` (latter four are stubs until 014/015).
- Single TOML config per collection (F-7.2). Schema includes: collection name, paths (cache, manifest, snapshots), Qdrant URL, embedder choice + model, adapter selections per content type.
- Structured progress output (F-7.4): one line per Document or stage transition, `tail -f` friendly.
- Exit codes: `0` clean, non-zero on real failure.
- `clap` for arg parsing.
- Composition root: `main` matches on adapter selections from config and constructs the concrete `Pipeline<...>` per arm — generics throughout, no `Box<dyn Trait>`.

## Test plan

- Integration: `librarian ingest --config foo.toml ./fixtures/` runs a small batch end-to-end against real Qdrant.
- `librarian remove <source_id>` followed by `librarian status` confirms removal.
- Bad config → human-readable error, non-zero exit.
